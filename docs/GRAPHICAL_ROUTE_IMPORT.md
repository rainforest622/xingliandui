# 图形路线导入与 WS63 本地巡检

## 当前实现

HarmonyOS App 提供两种路线来源：

- 手绘：在 300 x 190 路线画布中连续绘制，抬手后自动简化折线。
- AI 识图：从系统相册选择白底深色单线路线图，手机端完成灰度分割、主连通域提取、骨架化、路径追踪和折线简化。

两种来源最终都生成相同的路线模型，最多 12 段。图片本身不发送给 WS63，WS63 只接收压缩后的运动参数，因此不需要 Wi-Fi，也不会占用视频链路带宽。

## 最简参数

主界面只保留三个数值参数：

| 参数 | 范围 | 用途 |
| --- | --- | --- |
| 路线总长 | 0.5-200 m | 将图形中各段的相对长度换算为实际前进时间 |
| 速度 | 30-100 | WS63 电机速度档位 |
| 循环 | 0-20 | 0 表示连续；开放路线建议固定为 1 |

内部标定不占用主界面：满速每米按 `0.8s` 换算，90° 原地转向按最新实测 `0.686s` 换算。转角按图形向量自动计算，支持左转、右转和无需转向。

当前底盘没有编码器和定位模块，因此该功能属于“可校准时间轨迹”，不是 SLAM。场地或电量变化后，应先用 1 m 直线和 90° 转角重新验证标定。

## 星闪协议

路线采用事务式分包，只有全部分段收到后才更新当前路线：

```text
R0,<段数>,<速度>,<循环>                # 开始上传
R1,<序号>,<前进ms>,<方向>,<转向ms>     # 逐段上传
R2                                      # 校验并提交
P                                       # 启动巡检
S                                       # 立即停止
```

方向值：`0=不转`、`1=左转`、`2=右转`。App 等待每包应答，超时会重试两次；WS63 缺少任意分段时会拒绝 `R2`，旧路线不会被半包数据覆盖。

路线使用可校准的时间轨迹，而不是宣称有编码器闭环定位：手机按绘制的实际夹角计算每段转角，并按当前速度补偿直行与转向时长。页面可配置“满速秒/米”和“满速 90 度毫秒”，实车在不同地面、电池电量或负载下先做一次短距离校准即可。

WAVE ROVER 的树莓派桥有 `0.3s` 的运动安全看门狗。WS63 在任何非零运动期间每 `100ms` 重发当前左右轮速度 JSON，使连续直行、转弯和下一段衔接不会被桥误判为失联；停止、急停和传感器安全事件仍会立即输出零速度。

实车标定只需两次短测：以速度 `100` 让小车按 1 米直线运动，若实际距离为 `d` 米，则把“满速秒/米”改为 `旧值 / d`；再让小车转一次 90 度，若实际转角为 `a` 度，则把“满速 90 度毫秒”改为 `旧值 × 90 / a`。新参数会同时作用于手绘路线和图片识别路线，任意速度下均自动做速度补偿。

## 实机验收顺序

1. 烧录新固件，按 Reset，确认 `ws63_robot_mvp` 可以被手机扫描并连接。
2. 在 App 中恢复默认方形，路线长度先填小值，在空地以单圈低风险测试。
3. 点击“星闪导入”，确认状态依次显示分段接收并最终显示完整路线已保存。
4. 点击“启动巡检”，确认直行与转向自然衔接，停止键能立即打断。
5. 手绘一条含左转和右转的开放路线，循环设为 1，验证不同方向。
6. 选择一张白底深色粗线图，确认识别预览与原图一致后再下发。
