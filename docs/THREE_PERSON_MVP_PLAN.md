# 三人 MVP 工作计划

> 说明：本文件是立项阶段计划。当前真实完成状态和执行流程见 `docs/PROJECT_BASELINE_AND_WORKFLOW.md`。

## 目标与边界

项目按 2026 海思赛题的“星闪物联网应用方向”申报：WS63/WS63E + LiteOS 是主控，
SLE 承担关键控制，Wi-Fi 承担树莓派视频。树莓派视觉是增强项，不宣称满足海思
AI 端侧赛题对 SS928/Hi3516CV610 的平台要求。

第一阶段只追求一条可信、可测量的闭环：

1. 上位机发送 F/B/L/R/S；
2. WS63 执行双电机动作；
3. WS63 返回 ACK；
4. 上位机统计往返时延、抖动、丢包和急停响应；
5. 视频开启后重复测试，形成 SLE 与 Wi-Fi 控制的对比材料。

## 三人所有权

### 成员 A：WS63 与底盘（关键路径）

代码所有权：`ws63_liteos/`

本周交付：

- 跑通 WS63 官方例程、烧录和串口日志；
- 确认底板 PWM 发生器的官方驱动或 STM8 I2C 寄存器协议；
- 实现 `motor_apply(left, right)`，完成 F/B/L/R/S；
- 接入 500 ms 失联停车和前向避障停车；
- 输出引脚/总线表、串口日志和五动作视频。

验收：连续执行 100 轮 F-S-B-S-L-S-R-S 不复位、不失控；断开控制链路 500 ms
内停车。

### 成员 B：树莓派视频与轻量感知（并行路径）

代码所有权：`raspberry_pi/`

本周交付：

- 在树莓派运行摄像头采集；
- 浏览器打开 MJPEG 视频；
- 输出运动告警状态，后续可替换为轻量模型；
- 记录 640x480、10/15/20 FPS 下的带宽、CPU 和丢帧；
- 保存演示视频和截图。

验收：局域网连续推流 30 分钟；控制链路联调时视频端故障不阻塞底盘停车。

### 成员 C：协议、上位机、测试与材料（关键路径）

代码所有权：`upper_client/`、`simulator/`、`protocol_docs/`、`tests/`

本周交付：

- 维护 v1 协议，未经评审不改包长和字段语义；
- 将 TCP 替身换成 WS63 SLE/串口传输适配；
- 每条命令记录 seq、发送时间、ACK 时间、RTT 和状态；
- 输出空闲与视频拥塞两组实验数据；
- 每天归档日志、截图、问题和结论。

验收：不少于 1,000 个控制包；报告平均 RTT、P95、P99、最大值、抖动、丢包率和
急停响应。

## 合并规则

- A、B、C 只修改各自所有权目录；跨目录变更必须由目录负责人复核。
- `main` 只合并能运行测试或有真机日志证明的提交。
- 协议变更先改 `protocol_docs/protocol_v1.md`，再改双方代码。
- 每个问题必须附复现步骤、预期、实际和日志；禁止只写“不能用”。

## MVP 完成定义

- 自动化测试全部通过；
- 真机五动作与 ACK 通过；
- 失联和障碍两种情况都能进入安全停车；
- 视频可独立推流，视频故障不影响停车；
- 有基线/拥塞两组原始 CSV 和一页结论；
- 演示从冷启动到完成不超过 3 分钟。
